# Диагностика и проверка VPS сервера для Matrix

Этот документ описывает основные шаги по проверке состояния VPS сервера, выявлению ошибок и устранению сбоев при развертывании Matrix (Synapse).

## 1. Проверка системных ресурсов (ОЗУ, CPU, Диск)
Synapse требователен к ресурсам, особенно к оперативной памяти.

*   **Оперативная память и Swap:**
    ```bash
    free -h
    ```
    *Убедитесь, что есть свободная память. Если ОЗУ меньше 2 ГБ, обязательно должен быть настроен файл подкачки (Swap).*

*   **Проверка OOM (Out Of Memory):**
    Если процесс Synapse внезапно завершается, возможно, системе не хватает памяти:
    ```bash
    dmesg -T | grep -i oom
    journalctl -k | grep -i oom
    ```

*   **Свободное место на диске:**
    База данных и медиафайлы могут быстро занять весь диск.
    ```bash
    df -h
    ```

*   **Текущая нагрузка (CPU/RAM):**
    ```bash
    htop
    # или
    top
    ```

## 2. Сеть, порты и файрвол

*   **Проверка открытых портов:**
    Убедитесь, что сервер прослушивает нужные порты.
    ```bash
    ss -tulnp
    ```
    *Ожидаются порты 80, 443 (для Nginx/Traefik). Для федерации (если включена) — 8448.*

*   **Проверка файрвола (UFW):**
    ```bash
    ufw status
    ```
    *Должны быть разрешены соединения по портам 80/tcp и 443/tcp.*

*   **Проверка DNS записей:**
    Убедитесь, что домены указывают на IP вашего VPS.
    ```bash
    dig +short matrix.yourdomain.com
    dig +short element.yourdomain.com
    ```

## 3. Проверка работы Docker и контейнеров

*   **Статус службы Docker:**
    ```bash
    systemctl status docker
    ```

*   **Статус контейнеров Matrix:**
    Все контейнеры в стеке (synapse, db, nginx) должны иметь статус `Up`.
    ```bash
    docker compose ps
    ```
    *Если контейнер постоянно перезапускается (Restarting), проблема в его конфигурации.*

## 4. Чтение логов контейнеров
Основной инструмент диагностики — логи Docker. Запускайте команды в директории с `docker-compose.yml`.

*   **Логи Synapse:**
    ```bash
    docker compose logs -f synapse
    # Для фильтрации ошибок:
    docker compose logs synapse | grep -i error
    ```

*   **Логи базы данных (PostgreSQL):**
    Полезно, если Synapse не может подключиться к БД.
    ```bash
    docker compose logs -f db
    ```

*   **Логи Nginx (реверс-прокси):**
    Проверка ошибок 502/504 или проблем с SSL.
    ```bash
    docker compose logs -f nginx
    ```

## 5. Права доступа к файлам (Permissions)
Контейнер Synapse обычно работает от имени непривилегированного пользователя (UID 991). Если права на папку с данными сбиты, Synapse не запустится.

*   **Проверка прав директории с данными:**
    ```bash
    ls -la ./synapse-data
    ```
    *Если владелец `root`, Synapse не сможет писать в файлы базы (sqlite) или медиа. Восстановление (при необходимости):*
    ```bash
    sudo chown -R 991:991 ./synapse-data
    ```

## 6. SSL сертификаты
Проблемы с сертификатами не дадут клиенту подключиться к серверу.

*   **Проверка наличия и срока действия:**
    ```bash
    certbot certificates
    ```
*   **Проверка прав доступа к сертификатам:**
    Убедитесь, что процесс Nginx внутри Docker имеет доступ на чтение к `/etc/letsencrypt`.
